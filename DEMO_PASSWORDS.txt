Password And Signing Secret
---------------------------
Set `DEMO_AUTH_PASSWORD` and `DEMO_AUTH_SECRET` in the repo-root `.env` or
export them in your shell before deploy when you want fixed values.

If you leave either value blank, the canonical deploy script generates it and
prints the resulting demo password after rollout.

Endpoints
---------
Frontend and RTSP endpoints are deployment-specific. Use the URLs printed by
the deploy script or your platform ingress/load balancer output instead of
checking static endpoints into the repository.
